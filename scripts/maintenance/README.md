# 项目维护脚本

此目录包含**项目维护脚本**，用于维护工具集项目本身。

## 📋 脚本说明

### `cleanup.sh`

**用途**: 清理项目中的旧文件和过时内容

**使用场景**: 项目重构时，清理不再使用的文件

**使用方式**:
```bash
bash scripts/maintenance/cleanup.sh
```

**功能**:
- 移动旧文档到 `legacy/docs/`
- 移动旧配置到 `legacy/`
- 移动旧脚本到 `legacy/`
- 清理空目录

**注意**: 
- ⚠️ 此脚本会移动文件，请谨慎使用
- 建议先提交当前更改，再运行清理脚本
- 清理后的文件在 `legacy/` 目录中，不会丢失

## 🎯 使用场景

### 项目重构时

```bash
# 1. 提交当前更改
git add .
git commit -m "refactor: 准备清理旧文件"

# 2. 运行清理脚本
bash scripts/maintenance/cleanup.sh

# 3. 检查清理结果
ls -la legacy/

# 4. 提交清理结果
git add .
git commit -m "chore: 清理旧文件"
```

## 📝 注意事项

1. **项目特定**: 这些脚本只用于维护工具集项目本身
2. **不会输出**: 这些脚本不会输出到其他项目
3. **谨慎使用**: 清理脚本会移动文件，请确保已提交重要更改
